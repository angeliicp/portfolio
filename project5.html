Kaizen Management System

This project is a requirement to complete a project-based virtual internship with ID/X Partners and Rakamin Academy. It involved designing and implementing a data warehouse solution to integrate data from multiple sources, including database backup file (.bak), an Excel file, and a CSV file containing transaction data.

Goals

  • Integrate data from multiple sources to create a centralized database
  • Transform and cleanse data from duplication and inconsistencies
  • Design and implement star schema data model
  • Improve query performance by creating stored procedure

Technologies Used

  • Talend Open Studio
  • SQL Server Management Studio

Target Database Schema


Approach

Established connections from Talend to the source and target databases in SQL Server. This step is also part of configuring the metadata inputs in Talend together with the Excel files and CSV file.

Dimension Table ETL

  • DimBranch: Performed a direct mapping of columns from the source table to the target DimBranch table using the tMap component , followed by loading the data into the target database


  • DimCustomer: DimCustomer: Preprocessed the customer, city, and state tables by ensuring correct data types using tConvertType components. Then, joined the state, city, and customer table using tJoin component and loaded it into the SQL Server.


  • DimAccount: Performed column mapping using tMap component and defined foreign key constraint from the DimCustomer table using tDBRow component


    • Fact Table ETL

  • Converted the data types in the Excel file and CSV file columns to match with the transaction_db table
  • Consolidated data from the transaction_db table, Excel file, and CSV file into a single table using tUnite component
  • Eliminated duplicate data using the tUniqRow components
  • Defined foreign key constraint from relevant tables, like DimAccount and DimBranch, using the tDBRow Component


    • Stored Procedure SQL

    After completing the ETL processes, two stored procedures were created in SQL Server. The first procedure calculated the total transaction and its amount for each day to support daily transaction analysis. Then, the second procedure calculated the current account balance for each customer based on their transaction history.
